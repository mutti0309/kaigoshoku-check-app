# 介護食かんたんチェック診断

## 目的
家族介護者が、ご家族の食事のサイン（むせ・噛みにくさ・食事量の変化など）に気づき、見直しのきっかけを得るための小さなWebアプリ。医療判断ではなく「気づきの入口」として機能させる。

## 使い方
1. `index.html` をブラウザで開く（ダブルクリックでもOK）
2. 12個のチェック項目から、当てはまるものをタップ
3. 「診断する」を押すと、4タイプのいずれかの結果が表示される
4. 関連note記事への誘導カードが出る
5. 「もう一度チェックする」でリセット

公開する場合は、GitHub Pages・Cloudflare Pages・Netlifyなどに `index.html` 1枚をアップするだけで動く。

## 収益化導線
- **第1段階（現状）**：診断結果から関連note記事へ誘導 → noteの記事閲覧・サポート
- **第2段階（今後）**：診断タイプごとに、関連する介護食用品（とろみ剤・軟菜食用食材・栄養補助食品など）のアフィリエイトリンクを設置
- **第3段階**：診断結果をメール送付するフォームを追加 → メルマガリスト化

## 結果タイプ別 テストパターン（第3版）

| 出したいタイプ | チェックする項目 |
|---|---|
| まずは観察タイプ | チェックなし、または1個だけ |
| むせ込み・水分見直しタイプ | 食事中にむせる／水やお茶でむせやすい／食後に声がガラガラする |
| 噛みにくさ・まとまり見直しタイプ | 固いものを残す／肉や魚を噛みにくそう／きざみ食にしても食べにくそう |
| 飲み込みにくさタイプ | 食べ物が口の中に残る／パサつくものを嫌がる／薬が飲みにくそう |
| 食事量・栄養見直しタイプ | 食べる量が減ってきた／食事に時間がかかる／体重が減ってきた |
| 介助・環境見直しタイプ | 姿勢が崩れる／声かけしないと食べ進まない／途中で集中できない |
| 食事全体を見直すサインが重なっています（複合） | 合計5個以上、かつ4カテゴリ以上に分散<br>例：むせる／固いものを残す／口に残る／量が減った／姿勢が崩れる |
| 早めに専門職へ相談タイプ | urgent項目を2個以上<br>または urgent1個 + 他カテゴリ合計3個以上（総数4個以上） |

### 判定優先順
1. urgent条件 → 早めに専門職へ相談タイプ
2. 総チェック0〜1個 → まずは観察タイプ
3. 総チェック5個以上 かつ 4カテゴリ以上に分散 → 複合タイプ
4. それ以外 → 最高得点カテゴリ
5. 同点時 → cough > swallow > appetite > chew > environment

## 完成記録

### 2026-05-14 第3版 判定ロジック調整

「複合タイプ」が出やすすぎる問題を解消。

- 複合タイプを「合計5個以上 かつ 4カテゴリ以上に分散」の場合だけに限定（最後の逃げ道扱い）
- 2〜4個のチェックでは原則メインカテゴリを表示
- 複合タイプの表示名を「複合的に見直しタイプ」→「食事全体を見直すサインが重なっています」に変更
- タイプ名を読者にわかりやすく調整：
  - むせ込み注意タイプ → むせ込み・水分見直しタイプ
  - 噛みにくさタイプ → 噛みにくさ・まとまり見直しタイプ
  - 食欲低下タイプ → 食事量・栄養見直しタイプ
- urgent優先・観察フォールバック・同点時の優先順は据え置き

### 2026-05-14 第2版改善記録

判定ロジックを「チェック数」から「カテゴリ別スコア」に刷新。

- 質問を 12問 → 19問 に拡張し、6カテゴリ（cough / chew / swallow / appetite / environment / urgent）を付与
- 結果タイプを A/B/C/D の4種 → 7種（むせ込み / 噛みにくさ / 飲み込みにくさ / 食欲低下 / 介助・環境 / まずは観察 / 早めに専門職へ相談）に変更
- urgent項目（発熱・肺炎・急な体重減・呼吸困難）が一定以上で「早めに専門職へ相談タイプ」を優先表示
- 同点時の優先順を cough > swallow > appetite > chew > environment に固定
- 結果画面に「見えているサイン」「家庭でまずできること（3〜5個）」「注意したいこと」の3セクションを追加
- 「見えているサイン」は実際にチェックした項目を反映
- 表現を「可能性があります／きっかけになります／専門職に相談してください」に統一
- urgentタイプのみ警戒色（warn）で軽く強調表示
- チェックリストを6グループに分けて見出し付きで表示
- **noteUrl が `#` の結果タイプは関連noteリンクを非表示にする処理を追加**
- デザイン（茶系・ベージュ系、スマホ優先）は維持
